# Практическое занятие 24: GUI - паттерны проектирования

## Цель занятия
Изучить паттерны проектирования GUI-приложений, включая MVC, MVP, MVVM, а также паттерны, специфичные для графических интерфейсов, такие как Command, Observer, и другие.

## Задачи

### Задача 1: Паттерн MVC (20 баллов)
Реализуйте приложение с архитектурой Model-View-Controller:
- Создайте модель данных (Model) для управления списком задач
- Создайте представление (View) с использованием PyQt/Tkinter
- Создайте контроллер (Controller) для обработки пользовательских действий
- Обеспечьте разделение ответственностей

### Задача 2: Паттерн MVP (20 баллов)
Реализуйте приложение с архитектурой Model-View-Presenter:
- Модель без знаний о представлении
- Представление без бизнес-логики
- Презентер как посредник между моделью и представлением
- Четкое разделение слоев

### Задача 3: Паттерн Command (20 баллов)
Создайте систему команд для редактора:
- Абстрактный класс Command
- Конкретные команды (Add, Delete, Edit, Undo, Redo)
- История команд для возможности отмены/повтора
- Интерфейс для выполнения команд

### Задача 4: Паттерн Observer в GUI (20 баллов)
Реализуйте систему уведомлений с использованием паттерна Observer:
- Наблюдаемый объект (например, модель данных)
- Несколько наблюдателей (например, разные представления)
- Автоматическое обновление при изменении данных
- Гибкая система подписки

### Задача 5: Комплексное приложение (20 баллов)
Разработайте приложение "Заметки" с использованием нескольких паттернов:
- Используйте MVC или MVP архитектуру
- Реализуйте паттерн Command для операций с заметками
- Используйте Observer для обновления интерфейса при изменении данных
- Добавьте возможность Undo/Redo операций

## Методические указания
1. Используйте принципы разделения ответственностей
2. Обеспечьте слабую связанность между компонентами
3. Применяйте SOLID принципы при проектировании
4. Используйте абстрактные классы для определения интерфейсов паттернов

## Требования к отчету
- Исходный код приложений
- Диаграммы архитектуры
- Описание реализованных паттернов
- Сравнение разных подходов

## Критерии оценки
- Корректная реализация паттернов: 50%
- Качество архитектуры приложения: 30%
- Качество кода и документация: 20%
