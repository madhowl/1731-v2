# Практическое занятие 23: PyQt - продвинутые возможности

## Цель занятия
Изучить продвинутые возможности библиотеки PyQt, включая многопоточность в GUI-приложениях, создание пользовательских виджетов, работу с графикой и анимацией, а также продвинутые паттерны проектирования GUI.

## Задачи

### Задача 1: Пользовательские виджеты (20 баллов)
Создайте собственный виджет с использованием PyQt:
- Наследуйте от базового виджета (QWidget, QLabel, QPushButton и т.д.)
- Реализуйте собственные методы отрисовки
- Добавьте обработку событий
- Создайте виджет индикатора прогресса с анимацией

### Задача 2: Работа с графикой (20 баллов)
Реализуйте приложение с использованием QGraphicsView:
- Создайте сцену (QGraphicsScene) с различными элементами
- Добавьте возможность перемещения элементов
- Реализуйте масштабирование и прокрутку
- Добавьте элементы с обработкой событий

### Задача 3: Многопоточность в GUI (20 баллов)
Создайте приложение с использованием QThread:
- Выполнение длительных операций в отдельном потоке
- Обновление интерфейса из рабочего потока
- Использование сигналов для передачи данных между потоками
- Обработка завершения потока

### Задача 4: Анимация и эффекты (20 баллов)
Реализуйте анимацию с использованием QPropertyAnimation:
- Анимация перемещения элементов
- Анимация изменения свойств (цвет, размер, прозрачность)
- Сложные анимации с использованием QEasingCurve
- Анимация с использованием QTimer

### Задача 5: Комплексное приложение (20 баллов)
Разработайте приложение "Графический редактор":
- Используйте все изученные возможности
- Рисование различных фигур
- Выбор инструментов
- Сохранение и загрузка изображений
- Работа с несколькими слоями

## Методические указания
1. Используйте QPainter для пользовательской отрисовки
2. Применяйте QGraphicsView для сложной графики
3. Используйте QThread для многопоточности
4. Используйте QPropertyAnimation для анимации
5. Обрабатывайте исключения в потоках

## Требования к отчету
- Исходный код приложений
- Примеры пользовательских виджетов
- Описание архитектуры многопоточных приложений

## Критерии оценки
- Корректность работы приложений: 50%
- Использование продвинутых возможностей PyQt: 30%
- Качество архитектуры и документация: 20%
