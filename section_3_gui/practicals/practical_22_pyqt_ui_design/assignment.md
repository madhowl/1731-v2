# Практическое занятие 22: PyQt - проектирование UI

## Цель занятия
Изучить принципы проектирования пользовательского интерфейса в PyQt, включая создание сложных макетов, использование стилей, реализацию архитектуры MVC и разработку интуитивно понятных интерфейсов.

## Задачи

### Задача 1: Архитектура MVC (20 баллов)
Реализуйте приложение с архитектурой Model-View-Controller:
- Создайте модель данных для управления списком задач
- Создайте представление для отображения задач
- Создайте контроллер для управления взаимодействием

### Задача 2: Макеты и компоновка (20 баллов)
Создайте сложный интерфейс с использованием:
- Вложенных макетов (QVBoxLayout, QHBoxLayout, QGridLayout)
- QSplitter для разделения областей
- QStackedWidget для переключения между видами
- QTabWidget для организации информации

### Задача 3: Стилизация интерфейса (20 баллов)
Примените стилизацию к интерфейсу:
- Используйте QSS (Qt Style Sheets) для кастомизации
- Создайте темы оформления
- Реализуйте переключение между темами

### Задача 4: Кастомные виджеты (20 баллов)
Создайте собственные виджеты:
- Наследуйте от базовых классов PyQt
- Добавьте собственную логику и отображение
- Реализуйте собственные сигналы и слоты

### Задача 5: Комплексное приложение "Заметки" (20 баллов)
Разработайте приложение "Менеджер заметок":
- Используйте все изученные паттерны
- Реализуйте CRUD-операции для заметок
- Добавьте поиск и фильтрацию
- Обеспечьте сохранение данных

## Методические указания
1. Используйте архитектурные паттерны для организации кода
2. Применяйте подходящие макеты для разных ситуаций
3. Следите за доступностью и юзабилити интерфейса
4. Используйте сигналы и слоты для связи компонентов

## Требования к отчету
- Исходный код приложений
- Скриншоты интерфейсов
- Диаграммы архитектуры приложений

## Критерии оценки
- Корректность архитектуры MVC: 40%
- Качество интерфейса: 30%
- Использование стилей и кастомных виджетов: 20%
- Качество кода: 10%
