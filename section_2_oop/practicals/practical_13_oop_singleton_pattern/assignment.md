# Практическое занятие 13: ООП - паттерн Singleton

## Цель занятия
Изучить паттерн проектирования Singleton и научиться реализовывать его в Python различными способами, а также понять, когда и зачем его использовать.

## Задачи

### Задача 1: Базовая реализация Singleton (20 баллов)
Создайте класс `DatabaseConnection` с использованием паттерна Singleton:
- Реализуйте метод `__new__` для обеспечения единственности экземпляра
- Добавьте метод `connect()` для имитации подключения к базе данных
- Добавьте метод `disconnect()` для отключения

### Задача 2: Потокобезопасная реализация (20 баллов)
Реализуйте потокобезопасный Singleton с использованием:
- Блокировок (Lock) для предотвращения создания нескольких экземпляров в многопоточной среде
- Метода `getInstance()` для получения экземпляра класса

### Задача 3: Реализация через метакласс (20 баллов)
Создайте метакласс `SingletonMeta`, который:
- Обеспечивает паттерн Singleton для любого класса
- Примените его к классу `Logger` для ведения логов

### Задача 4: Декоратор Singleton (20 баллов)
Создайте декоратор `singleton`, который:
- Преобразует любой класс в Singleton
- Примените его к классу `ConfigurationManager` для управления настройками приложения

### Задача 5: Применение Singleton в реальном сценарии (20 баллов)
Реализуйте систему уведомлений:
- Класс `NotificationManager` как Singleton
- Методы для отправки различных типов уведомлений
- Очередь уведомлений и возможность их просмотра

## Методические указания
1. Используйте разные подходы к реализации Singleton (через `__new__`, метаклассы, декораторы)
2. Обратите внимание на потокобезопасность при многопоточном использовании
3. Рассмотрите плюсы и минусы использования паттерна Singleton
4. Следите за тем, чтобы Singleton действительно обеспечивал уникальность экземпляра

## Требования к отчету
- Исходный код всех реализаций Singleton
- Примеры использования каждого подхода
- Сравнение различных реализаций и их применимость
