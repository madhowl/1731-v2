# Практическое занятие 14: ООП - паттерн Factory

## Цель занятия
Изучить паттерн проектирования Factory (Фабрика) и научиться реализовывать его в Python для создания объектов без указания их конкретных классов.

## Задачи

### Задача 1: Простая фабрика (20 баллов)
Создайте фабрику `VehicleFactory` для создания транспортных средств:
- Абстрактный класс `Vehicle` с методом `drive()`
- Конкретные классы: `Car`, `Motorcycle`, `Truck`
- Фабричный метод `create_vehicle(vehicle_type)`

### Задача 2: Фабричный метод (20 баллов)
Реализуйте паттерн "Фабричный метод" с:
- Абстрактным классом `DocumentCreator`
- Конкретными классами: `PDFDocumentCreator`, `WordDocumentCreator`, `ExcelDocumentCreator`
- Абстрактным методом `create_document()` и конкретными реализациями

### Задача 3: Абстрактная фабрика (20 баллов)
Создайте абстрактную фабрику UI-элементов:
- Интерфейс `UIFactory` с методами для создания кнопок и текстовых полей
- Конкретные фабрики: `WindowsUIFactory`, `MacUIFactory`, `LinuxUIFactory`
- Классы элементов: `Button`, `TextField`

### Задача 4: Параметризованная фабрика (20 баллов)
Реализуйте фабрику `ShapeFactory` с параметрами:
- Возможность создания фигур с определенными свойствами (цвет, размер)
- Поддержка различных типов фигур: `Circle`, `Rectangle`, `Triangle`
- Валидация параметров при создании

### Задача 5: Фабрика с кэшированием (20 баллов)
Создайте фабрику `ObjectPoolFactory`:
- Кэширование созданных объектов для повторного использования
- Методы получения и возврата объектов
- Управление жизненным циклом объектов

## Методические указания
1. Используйте абстрактные классы для определения общего интерфейса
2. Применяйте принципы SOLID при проектировании фабрик
3. Обеспечьте гибкость и расширяемость системы
4. Обрабатывайте ошибки при создании объектов

## Требования к отчету
- Исходный код всех реализаций фабрик
- Примеры использования каждой фабрики
- Сравнение различных подходов к реализации

## Критерии оценки
- Корректная реализация паттерна Factory: 50%
- Понимание различных вариантов применения: 30%
- Качество кода и документация: 20%
