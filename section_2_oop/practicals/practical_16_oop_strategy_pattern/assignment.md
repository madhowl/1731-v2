# Практическое занятие 16: ООП - паттерн Strategy

## Цель занятия
Изучить паттерн проектирования Strategy (Стратегия) и научиться реализовывать его в Python для выбора алгоритмов во время выполнения программы.

## Задачи

### Задача 1: Базовая реализация Strategy (20 баллов)
Создайте систему сортировки с использованием паттерна Strategy:
- Интерфейс `SortingStrategy` с методом `sort()`
- Конкретные стратегии: `BubbleSort`, `QuickSort`, `MergeSort`
- Класс `Sorter` как контекст, использующий стратегию

### Задача 2: Финансовые стратегии (20 баллов)
Реализуйте систему расчета налогов:
- Интерфейс `TaxStrategy` с методом `calculate_tax()`
- Конкретные стратегии: `ProgressiveTax`, `FlatTax`, `ProportionalTax`
- Класс `TaxCalculator` как контекст

### Задача 3: Стратегии с параметрами (20 баллов)
Создайте систему сжатия данных:
- Интерфейс `CompressionStrategy` с методом `compress()`
- Конкретные стратегии: `ZipCompression`, `RarCompression`, `GzipCompression`
- Возможность передачи параметров сжатия в стратегии

### Задача 4: Динамическое изменение стратегии (20 баллов)
Реализуйте систему маршрутизации:
- Интерфейс `RouteStrategy` с методом `find_route()`
- Конкретные стратегии: `FastestRoute`, `ShortestRoute`, `EcoRoute`
- Возможность изменения стратегии во время выполнения

### Задача 5: Комбинированные стратегии (20 баллов)
Создайте систему обработки данных:
- Комбинация нескольких стратегий (например, сначала сортировка, потом фильтрация)
- Класс `CompositeStrategy` для комбинирования стратегий
- Возможность создания сложных алгоритмов из простых

## Методические указания
1. Используйте абстрактные классы для определения интерфейсов стратегий
2. Обеспечьте возможность легкой замены стратегий
3. Рассмотрите использование фабрик для создания стратегий
4. Обрабатывайте ошибки при выборе стратегии

## Требования к отчету
- Исходный код всех реализаций Strategy
- Примеры использования каждой стратегии
- Сравнение эффективности различных стратегий

## Критерии оценки
- Корректная реализация паттерна Strategy: 50%
- Понимание различных вариантов применения: 30%
- Качество кода и документация: 20%
