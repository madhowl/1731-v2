# Практическое занятие 15: ООП - паттерн Observer

## Цель занятия
Изучить паттерн проектирования Observer (Наблюдатель) и научиться реализовывать его в Python для создания системы подписки на события и уведомления об изменениях.

## Задачи

### Задача 1: Базовая реализация Observer (20 баллов)
Создайте систему наблюдения за курсом валют:
- Интерфейс `Observer` с методом `update()`
- Интерфейс `Subject` с методами для управления подписчиками
- Класс `CurrencyRate` как наблюдаемый объект
- Классы наблюдателей: `EmailNotifier`, `SMSNotifier`

### Задача 2: Использование встроенных средств (20 баллов)
Реализуйте систему уведомлений с использованием:
- Событийной модели Python
- Механизма callback-функций
- Класса `EventManager` для управления событиями

### Задача 3: Параметризованные уведомления (20 баллов)
Создайте систему уведомлений с фильтрацией:
- Возможность подписки на определенные типы событий
- Передача данных о событии подписчикам
- Класс `EventPublisher` и `EventSubscriber`

### Задача 4: Асинхронный Observer (20 баллов)
Реализуйте асинхронную систему наблюдения:
- Использование asyncio для асинхронных уведомлений
- Класс `AsyncSubject` и `AsyncObserver`
- Обработка уведомлений в отдельных задачах

### Задача 5: Практическое применение (20 баллов)
Создайте систему уведомлений для социальной сети:
- Класс `UserProfile` как наблюдаемый объект
- Наблюдатели: `FriendNotifier`, `FeedUpdater`, `NotificationService`
- Уведомления о действиях пользователя

## Методические указания
1. Используйте абстрактные классы для определения интерфейсов
2. Обеспечьте гибкость системы через параметры уведомлений
3. Обрабатывайте ошибки при уведомлении подписчиков
4. Рассмотрите асинхронные подходы для масштабируемых систем

## Требования к отчету
- Исходный код всех реализаций Observer
- Примеры использования каждого подхода
- Сравнение синхронных и асинхронных реализаций

## Критерии оценки
- Корректная реализация паттерна Observer: 50%
- Понимание различных вариантов применения: 30%
- Качество кода и документация: 20%
