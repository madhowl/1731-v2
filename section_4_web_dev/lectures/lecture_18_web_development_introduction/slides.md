## 1. Клиент-серверная архитектура

Клиент-серверная архитектура — это распределённая вычислительная архитектура, в которой задачи разделены между поставщиками услуг (серверами) и заказчиками услуг (клиентами).

### Основные компоненты:
- **Клиент** - программа, которая отправляет запросы на сервер
- **Сервер** - программа, которая принимает запросы от клиентов и отвечает на них
- **Сеть** - канал связи между клиентом и сервером

### Преимущества:
- Разделение ответственности
- Масштабируемость
- Централизованное управление данными

## 2. Протокол HTTP

HTTP (HyperText Transfer Protocol) — протокол передачи гипертекста, используется для загрузки веб-страниц, изображений, видео и других ресурсов в сети Интернет.

### Основные характеристики:
- Протокол прикладного уровня
- Работает поверх TCP/IP
- Не сохраняет состояние между запросами (stateless)

### Структура HTTP-запроса:
```
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
Connection: close
```

### Структура HTTP-ответа:
```
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 137
Connection: close

<!DOCTYPE html>
<html>
<head><title>Пример страницы</title></head>
<body><h1>Привет, мир!</h1></body>
</html>
```

## 3. Основные методы HTTP

| Метод | Назначение |
|------------|
| GET | Запрос данных с сервера |
| POST | Отправка данных на сервер |
| PUT | Обновление данных на сервере |
| DELETE | Удаление данных с сервера |
| PATCH | Частичное обновление данных |

### Коды состояния HTTP:
- **1xx** - информационные сообщения
- **2xx** - успешное выполнение (200 OK)
- **3xx** - перенаправление (301 Moved Permanently)
- **4xx** - ошибка клиента (404 Not Found)
- **5xx** - ошибка сервера (500 Internal Server Error)

## 4. Структура веб-приложений

### Типичная архитектура:
```
[Клиент] <---> [Веб-сервер] <---> [База данных]
```

### Технологии:
- **Frontend**: HTML, CSS, JavaScript
- **Backend**: Python (Flask, Django), Node.js, PHP
- **Базы данных**: MySQL, PostgreSQL, MongoDB
