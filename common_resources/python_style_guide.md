# Руководство по стилю кода Python

## PEP 8 - стиль программирования на Python

### Именование
- `class_names`: Классы именуются в формате CapWords (PascalCase)
- `function_names`, `variable_names`: Функции и переменные используют lowercase с подчеркиваниями
- `CONSTANT_NAMES`: Константы именуются заглавными буквами с подчеркиваниями
- `_private_names`: Приватные атрибуты начинаются с подчеркивания

### Отступы
- Используйте 4 пробела на один уровень отступа
- Не используйте табуляцию

### Длина строки
- Ограничьте длину строк максимум 79 символами
- Для комментариев ограничьте длину до 72 символов

### Пустые строки
- Оставляйте 2 пустые строки перед определением класса или функции верхнего уровня
- Оставляйте 1 пустую строку внутри определения класса между методами

### Импорты
- Каждый импорт должен быть на отдельной строке
- Импорты всегда помещаются в начале файла
- Порядок: стандартная библиотека, затем сторонние библиотеки, затем локальные импорты
- Не используйте `from module import *`

### Комментарии
- Все комментарии пишутся на английском языке
- Блок комментариев начинается с заглавной буквы и заканчивается точкой
- Inline-комментарии отделяются двумя пробелами от инструкции

### Docstrings
- Используйте docstrings для всех публичных модулей, функций, классов и методов
- Для однострочных docstrings используйте двойные кавычки

### Примеры оформления

```python
"""Пример оформления модуля в соответствии с PEP 8."""

import os
import sys

from third_party import lib1, lib2


class MyClass:
    """Пример оформления класса."""
    
    def __init__(self, param1, param2=None):
        """Инициализирует экземпляр класса.
        
        Args:
            param1 (str): Первый обязательный параметр
            param2 (int, optional): Второй опциональный параметр
        """
        self.param1 = param1
        self.param2 = param2
        
    def my_method(self):
        """Пример метода класса."""
        return self.param1
```

## Дополнительные рекомендации

### Аннотации типов
Используйте аннотации типов для лучшей читаемости:

```python
def greet(name: str) -> str:
    """Приветствует пользователя по имени.
    
    Args:
        name: Имя пользователя
        
    Returns:
        Приветственное сообщение
    """
    return f"Hello, {name}!"
```

### Использование f-строк
Предпочтительно использовать f-строки для форматирования:

```python
name = "Alice"
age = 30
message = f"Name: {name}, Age: {age}"
```

### Организация кода
- Разделяйте логические блоки кода пустыми строками
- Используйте осмысленные имена переменных
- Следите за тем, чтобы функции выполняли только одну задачу
- Избегайте глубокой вложенности (максимум 3 уровня)
