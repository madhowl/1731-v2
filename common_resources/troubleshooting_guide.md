# Руководство по устранению неполадок

## Общие проблемы и решения

### Проблемы с установкой пакетов
**Проблема:** Ошибка при установке пакета с помощью pip
**Решение:** 
- Убедитесь, что используете правильную версию Python
- Попробуйте обновить pip: `python -m pip install --upgrade pip`
- Используйте виртуальное окружение

### Проблемы с виртуальными окружениями
**Проблема:** Не удается активировать виртуальное окружение в PowerShell
**Решение:** 
- В PowerShell выполните: `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`
- Затем активируйте окружение: `.\venv\Scripts\Activate.ps1`

### Проблемы с импортом модулей
**Проблема:** Ошибка `ModuleNotFoundError`
**Решение:**
- Проверьте, установлен ли требуемый пакет
- Убедитесь, что структура проекта корректна
- Используйте относительные импорты при необходимости

## Проблемы в GUI-приложениях

### Tkinter
**Проблема:** Приложение не отвечает или зависает
**Решение:**
- Убедитесь, что вызов `mainloop()` происходит только один раз
- Не выполняйте долгие операции в основном потоке

**Проблема:** Виджеты не отображаются
**Решение:**
- Проверьте, что используется один менеджер размещения (pack, grid, place) в одном родительском элементе
- Убедитесь, что виджеты созданы и не удалены сборщиком мусора

### PyQt
**Проблема:** Ошибка при импорте PyQt
**Решение:**
- Установите PyQt5 или PyQt6: `pip install PyQt5`
- Проверьте совместимость версии Python

## Проблемы с веб-разработкой

### Flask
**Проблема:** Ошибка при запуске Flask-приложения
**Решение:**
- Убедитесь, что переменная окружения FLASK_APP установлена правильно
- Проверьте, что все зависимости установлены

**Проблема:** Шаблоны не находятся
**Решение:**
- Убедитесь, что папка templates находится в правильном месте
- Проверьте регистр имен файлов

### Django
**Проблема:** Ошибка при миграции базы данных
**Решение:**
- Выполните `python manage.py makemigrations`
- Затем `python manage.py migrate`

## Проблемы с базами данных

### SQLite
**Проблема:** База данных заблокирована
**Решение:**
- Убедитесь, что соединение с базой данных закрывается после использования
- Проверьте, что нет одновременных попыток записи

### SQLAlchemy
**Проблема:** Ошибки при работе с сессиями
**Решение:**
- Используйте контекстные менеджеры или явно вызывайте `session.close()`
- Убедитесь, что используется `session.commit()` для сохранения изменений

## Проблемы с Git

**Проблема:** Конфликты при слиянии
**Решение:**
- Разрешите конфликты вручную
- Используйте `git merge-tool` для визуального редактирования
- После разрешения выполните `git add` и `git commit`

**Проблема:** Случайно удаленные файлы
**Решение:**
- Восстановите файлы из предыдущего коммита: `git checkout HEAD~1 filename`
- Или восстановите из stash: `git stash pop`

## Отладка

### Использование отладчика
- В Python используйте `pdb`: `import pdb; pdb.set_trace()`
- В IDE используйте встроенные средства отладки

### Логирование
```python
import logging

logging.basicConfig(level=logging.DEBUG)
logger = logging.getLogger(__name__)

logger.debug("Отладочное сообщение")
logger.info("Информационное сообщение")
logger.warning("Предупреждение")
logger.error("Сообщение об ошибке")
```

## Наиболее частые ошибки синтаксиса

1. Пропущенные двоеточия после условий и циклов
2. Неправильная индентация
3. Несоответствие скобок
4. Неправильное использование ключевых слов
5. Опечатки в именах переменных и функций
