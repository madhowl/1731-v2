# Домашнее задание 6: Финальный проект

## Цель задания
Создать комплексное прикладное приложение, объединяющее все изученные темы в рамках дисциплины "Разработка прикладных приложений". Это задание является подготовкой к курсовому проекту.

## Задачи

### Задача 1: Планирование и архитектура (20 баллов)
Разработайте архитектуру приложения, которое будет включать:
- Объектно-ориентированную структуру
- Графический интерфейс пользователя (Tkinter или PyQt)
- Веб-компонент (Flask или Django)
- Работу с базой данных (SQLAlchemy)
- Интеграцию с внешними API
- Механизмы безопасности

Подготовьте диаграмму классов и схему базы данных.

### Задача 2: Реализация модели (20 баллов)
Создайте модельную часть приложения:
- Реализуйте основные классы предметной области
- Создайте модели данных для хранения информации
- Реализуйте валидацию данных
- Обеспечьте безопасное хранение конфиденциальной информации

### Задача 3: GUI-интерфейс (15 баллов)
Разработайте графический интерфейс для основного функционала:
- Создайте главное окно приложения
- Реализуйте основные экраны/страницы
- Обеспечьте навигацию между экранами
- Добавьте элементы ввода и отображения данных

### Задача 4: Веб-интерфейс (15 баллов)
Реализуйте веб-часть приложения:
- Создайте REST API для основного функционала
- Реализуйте веб-страницы для отображения данных
- Обеспечьте интеграцию с базой данных
- Добавьте аутентификацию пользователей

### Задача 5: Интеграция компонентов (15 баллов)
Объедините все компоненты в единое приложение:
- Обеспечьте взаимодействие между GUI и веб-компонентами
- Реализуйте синхронизацию данных между интерфейсами
- Обеспечьте согласованную работу всех частей приложения

### Задача 6: Безопасность и тестирование (15 баллов)
- Добавьте защиту от основных уязвимостей (XSS, CSRF, SQL-инъекции)
- Напишите модульные тесты для основных функций
- Реализуйте логирование работы приложения
- Обеспечьте обработку ошибок и исключений

## Требования к оформлению

1. Все функции и классы должны иметь docstring
2. Код должен соответствовать PEP 8
3. Добавьте аннотации типов
4. Приложение должно быть разделено на логические модули
5. Добавьте файл requirements.txt с зависимостями
6. Подготовьте краткую документацию по установке и запуску

## Срок сдачи
Через 3 недели с момента выдачи задания

## Критерии оценки
- Качество архитектуры приложения: 20%
- Функциональность и интеграция компонентов: 30%
- Безопасность и надежность: 25%
- Качество кода и документации: 15%
- Следование стандартам кодирования: 10%
