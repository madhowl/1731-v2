# Руководство по сдаче курсового проекта

## Общие положения

Курсовой проект является завершающим этапом изучения дисциплины "Разработка прикладных приложений". Проект должен быть представлен в виде исходного кода, документации и презентации к защите.

## Сроки сдачи

- **Промежуточная защита**: Через 2 недели после назначения проекта
- **Финальная сдача исходного кода**: За неделю до защиты
- **Защита проекта**: В последние 2 недели семестра

## Форма представления

Курсовой проект должен быть представлен в следующем виде:

1. **Исходный код** в виде репозитория на GitHub
2. **Документация** в формате README.md в корне проекта
3. **Презентация** к защите (PDF или PowerPoint)

## Требования к репозиторию

- Репозиторий должен иметь понятное название, отражающее суть проекта
- В корне репозитория обязательно наличие файла README.md
- Код должен быть разбит на логические модули
- Используйте ветвление: основной код в ветке main/master, разработка в feature-ветках
- Регулярно делайте коммиты с осмысленными сообщениями

## Структура проекта

```
course-project/
├── src/
│   ├── models/          # Модельный слой
│   ├── views/           # Представления (GUI или Web)
│   ├── controllers/     # Контроллеры
│   ├── utils/           # Вспомогательные функции
│   ├── static/          # Статические файлы (если есть)
│   ├── templates/       # Шаблоны (если есть)
│   ├── tests/           # Тесты
│   └── main.py          # Главный файл запуска
├── docs/                # Документация
├── database/            # Файлы базы данных (если применимо)
├── config/              # Конфигурационные файлы
├── requirements.txt     # Зависимости Python
├── README.md            # Основная документация
└── .gitignore           # Файлы, игнорируемые Git
```

## Требования к документации

Файл README.md должен содержать:

1. **Описание проекта** - краткое описание функциональности
2. **Технологии** - список используемых технологий и библиотек
3. **Установка и запуск** - пошаговая инструкция по установке и запуску
4. **Использование** - описание работы с приложением
5. **Архитектура** - описание архитектурных решений
6. **Авторы** - информация о разработчиках

## Требования к презентации

Презентация должна содержать:

1. **Титульный слайд** - название проекта, авторы, дата
2. **Описание задачи** - какую проблему решает приложение
3. **Архитектура приложения** - диаграммы и описание структуры
4. **Демонстрация функционала** - скриншоты или живая демонстрация
5. **Технологии** - используемые технологии и библиотеки
6. **Проблемы и решения** - с какими трудностями сталкивались
7. **Выводы** - что было изучено, что можно улучшить

## Критерии приемки

Перед сдачей проекта убедитесь, что:

- [ ] Приложение запускается без ошибок
- [ ] Все зависимости указаны в requirements.txt
- [ ] Код соответствует PEP 8
- [ ] Реализованы все требуемые функции
- [ ] Документация полная и понятная
- [ ] Код размещен в публичном репозитории
- [ ] Презентация готова к защите

## Формат сдачи

1. **За неделю до защиты**:
   - Отправьте ссылку на репозиторий преподавателю
   - Убедитесь, что репозиторий публичный
   - Проверьте, что последний коммит содержит рабочую версию

2. **В день защиты**:
   - Подготовьте презентацию
   - Будьте готовы ответить на вопросы по коду
   - Проведите демонстрацию работы приложения

## Часто задаваемые вопросы

**Q: Можно ли использовать внешние библиотеки?**
A: Да, использование внешних библиотек поощряется, но важно понимать, как они работают.

**Q: Какой объем должен быть у проекта?**
A: Проект должен демонстрировать знания по всем разделам курса, но не быть чрезмерно сложным.

**Q: Можно ли делать проект в команде?**
A: Курсовой проект обычно выполняется индивидуально, но по согласованию с преподавателем возможна групповая работа.

**Q: Что делать, если не успеваю к сроку?**
A: Обязательно сообщите преподавателю заранее, чтобы обсудить возможные варианты.
